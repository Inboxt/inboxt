# Contributing to Inboxt

Thank you for your interest in contributing to Inboxt! We are happy that you are looking to help improve the project.

## Ways to contribute

1. **Bug Reports & Feature Requests:** If you find a bug or have an idea, please [open an issue](https://github.com/Inboxt/inboxt/issues) or start a [discussion](https://github.com/Inboxt/inboxt/discussions).
2. **Code:** Bug fixes, small improvements, and even major features are welcome.
3. **Documentation:** Documentation is just as important as code. Improving guides or fixing typos is always appreciated.

## Getting started with development

For detailed instructions on setting up your local environment, please refer to our **[Local Development Guide](https://docs.inboxt.app/local-development)**.

### Quick Start (Local Development)

```bash
# 1. Clone and enter the repo
git clone https://github.com/Inboxt/inboxt.git
cd inboxt

# 2. Setup environment
cp dev.env.example .env

# 3. Install dependencies
npm install

# 4. Start infrastructure (Docker) and Web App (Vite)
npm run dev
```

## Pull Request Process

1. **Fork and Branch:** Fork the repository and create a new branch for your change.
2. **Commit:** Write clear, descriptive commit messages.
3. **Open a PR:** Once you’re ready, open a Pull Request.

For a deeper dive into our architecture, directory structure, and development workflows, see the full **[Contributing Guide](https://docs.inboxt.app/contributing)**.
